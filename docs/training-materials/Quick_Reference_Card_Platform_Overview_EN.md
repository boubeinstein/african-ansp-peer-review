## Self-Assessment Workflow

```
┌──────────┐    ┌─────────────┐    ┌───────────┐    ┌───────────┐
│  DRAFT   │───▶│ IN_PROGRESS │───▶│ SUBMITTED │───▶│ COMPLETED │
└──────────┘    └─────────────┘    └───────────┘    └───────────┘
```

**Steps:**
1. Create new assessment (ANS or SMS)
2. Answer protocol questions
3. Review EI scores and maturity levels
4. Submit for review
5. Mark as complete after verification

## Peer Review Phases

```
REQUESTED → APPROVED → PLANNING → FIELDWORK → REPORTING → COMPLETED
```

| Phase | Activities |
|-------|------------|
| **Requested** | Host submits review request |
| **Approved** | Coordinator approves request |
| **Planning** | Team assignment, document request |
| **Fieldwork** | On-site activities, interviews |
| **Reporting** | Draft report, findings entry |
| **Completed** | Final report, CAP follow-up |

---

## Fieldwork Checklist (14 Items)

### Pre-Visit (4 items)
- [ ] Document request sent
- [ ] Pre-visit documents received
- [ ] Coordination meeting held
- [ ] Review plan approved

### On-Site (6 items)
- [ ] Opening meeting conducted
- [ ] Staff interviews completed
- [ ] Facilities inspection completed
- [ ] Document review completed
- [ ] Findings discussed with host
- [ ] Closing meeting conducted

### Post-Visit (4 items)
- [ ] All findings entered
- [ ] Evidence uploaded
- [ ] Draft report prepared
- [ ] Host feedback received

**Note:** Items with a lock icon require document upload to complete

## Document Categories

| Category | When Used |
|----------|-----------|
| Pre-Visit Request | Document request letter |
| Host Submission | Documents from host ANSP |
| Interview Notes | Staff interview records |
| Evidence | Photos, inspection reports |
| Draft Report | Preliminary review report |
| Final Report | Completed review report |
| CAP Evidence | Corrective action proof |
| Correspondence | Email communications |

### Document Status Flow

```
UPLOADED → UNDER_REVIEW → REVIEWED → APPROVED
                ↓
        REJECTED (requires re-upload)
```

## Finding Severity Levels

| Level | Indicator | Response Time |
|-------|-----------|---------------|
| **Critical** | Red | Immediate action |
| **High** | Orange | 30 days |
| **Medium** | Yellow | 60 days |
| **Low** | Green | 90 days |

### Finding Types
- **Non-Conformity** - Does not meet requirements
- **Concern** - Potential non-conformity
- **Observation** - Improvement opportunity

---

## CAP Workflow

```
DRAFT → SUBMITTED → ACCEPTED → IMPLEMENTED → VERIFIED → CLOSED
              ↓
        REJECTED (requires revision)
```

**Required Fields:**
- Root cause analysis
- Proposed corrective actions
- Due date
- Responsible person
