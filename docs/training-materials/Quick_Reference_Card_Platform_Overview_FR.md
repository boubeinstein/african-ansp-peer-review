## Flux d'Auto-evaluation

```
┌───────────┐    ┌────────────┐    ┌──────────┐    ┌──────────┐
│ BROUILLON │───▶│ EN_COURS   │───▶│ SOUMIS   │───▶│ TERMINE  │
└───────────┘    └────────────┘    └──────────┘    └──────────┘
```

**Etapes:**
1. Creer une nouvelle evaluation (ANS ou SMS)
2. Repondre aux questions du protocole
3. Examiner les scores EI et niveaux de maturite
4. Soumettre pour revision
5. Marquer comme termine apres verification

---

## Phases de la Revue par les Pairs

```
DEMANDE → APPROUVE → PLANIFICATION → TERRAIN → RAPPORT → TERMINE
```

| Phase | Activites |
|-------|-----------|
| **Demande** | L'hote soumet la demande de revue |
| **Approuve** | Le coordinateur approuve la demande |
| **Planification** | Affectation d'equipe, demande de documents |
| **Terrain** | Activites sur site, entretiens |
| **Rapport** | Rapport preliminaire, saisie des constatations |
| **Termine** | Rapport final, suivi des PAC |

---

## Liste de Controle du Terrain (14 Elements)

### Pre-visite (4 elements)
- [ ] Demande de documents envoyee
- [ ] Documents pre-visite recus
- [ ] Reunion de coordination tenue
- [ ] Plan de revue approuve

### Sur Site (6 elements)
- [ ] Reunion d'ouverture tenue
- [ ] Entretiens du personnel termines
- [ ] Inspection des installations terminee
- [ ] Examen des documents termine
- [ ] Constatations discutees avec l'hote
- [ ] Reunion de cloture tenue

### Post-visite (4 elements)
- [ ] Toutes les constatations saisies
- [ ] Preuves telechargees
- [ ] Rapport preliminaire prepare
- [ ] Commentaires de l'hote recus

**Note:** Les elements avec une icone de cadenas necessitent un telechargement de document

## Niveaux de Gravite des Constatations

| Niveau | Indicateur | Delai de Reponse |
|--------|------------|------------------|
| **Critique** | Rouge | Action immediate |
| **Eleve** | Orange | 30 jours |
| **Moyen** | Jaune | 60 jours |
| **Faible** | Vert | 90 jours |

### Types de Constatations
- **Non-conformite** - Ne repond pas aux exigences
- **Preoccupation** - Non-conformite potentielle
- **Observation** - Opportunite d'amelioration

---

## Flux du PAC

```
BROUILLON → SOUMIS → ACCEPTE → MIS EN OEUVRE → VERIFIE → CLOTURE
                ↓
        REJETE (necessite revision)
```

**Champs Obligatoires:**
- Analyse des causes profondes
- Actions correctives proposees
- Date d'echeance
- Personne responsable
